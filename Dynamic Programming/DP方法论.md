# 动态规划DP总结

## 状态转移的基本模式
1. 线形转移：`dp[i] = f(dp[i - 1], dp[i - 2]...)`
   1. 适用：斐波那契数列，爬楼梯，解码方式
   2. Leetcode 62, 63, 64, 300
2. 决策转移：`dp[i] = min/max(选择1， 选择2)`
   1. 适用：背包问题，路径选择，资源分配
   2. Leetcode 322, 416, 494，337
3. 匹配转移：`dp[i] = f[dp[i - 1][j], dp[i][j -1], dp[i - 1][j - 1]]`
   1. 适用：字符串匹配，编辑距离，LCS
   2. Leetcode：1143，44，10
4. 区间合并:`dp[left][right] = f(dp[left][k], dp[k + 1][right])`
   1. Note: 最外层循环length，中间层循环起始index i，j = i+ length，最内层循环k + 1到j + 1
   2. 适用：回文，合并成本
   3. Leetcode: 312, 516, 1000, 375

## 动态规划的初始化和边界处理
1. 空集/空状态的处理: `dp[0]/dp[0][0]`定义空状态
2. 无效状态标记：`float('inf')或-1`表示状态不可达

## 动态规划/DP问题的解题框架
1. 状态定义：`dp[i][j]/dp[i]`的含义
2. 初始化：最小问题的解
3. 计算顺序：确认状态的依赖关系，确保依赖的状态先计算
4. 状态转移：基于已解决的子问题推导当前状态
5. 答案提取：从最终状态获取答案
